# 꿀벌 수 측정 앱

벌통에서 꺼낸 소비 한 면을 스마트폰으로 촬영하면 일벌 개체수를 추정하고, 여러 소비 면을 누적해 한 벌통의 일벌 수를 기록하는 PWA 프로토타입입니다.

## 실행 방법

이 폴더의 `index.html`을 브라우저로 열면 바로 사용할 수 있습니다. 스마트폰에서 앱처럼 설치하거나 카메라 권한과 오프라인 캐시를 안정적으로 쓰려면 로컬 또는 배포 서버에서 열어야 합니다.

## 현재 기능

- 소비 사진 촬영 또는 업로드
- 소비 번호와 앞면/뒷면 기록
- 사진 기반 일벌 수 즉시 추정
- 벌통 전체 합계, 촬영 면 수, 평균 신뢰도 표시
- 소비별 결과 삭제
- CSV 내보내기와 요약 공유
- 브라우저 로컬 저장소에 임시 기록 보관

## 현재 분석 방식의 한계

현재 버전은 사진에서 일벌로 보이는 어두운 영역, 연결된 덩어리 수, 소비 면적 대비 점유율을 계산해 추정합니다. 현장 테스트용 MVP로는 유용하지만, 연구용 절대 정확도를 보장하려면 실제 소비 사진으로 학습한 객체 탐지 또는 세그멘테이션 모델이 필요합니다.

## 다음 단계에서 필요한 자료

1. 같은 조건으로 촬영한 소비 사진 300-1,000장 이상
2. 사진별 실제 일벌 개체수 또는 벌 위치 박스/점 라벨
3. 벌집 종류, 소비 크기, 촬영 거리, 조명 조건 기록
4. 모델 검증용으로 따로 보관할 사진 15-20%

## 권장 개발 방향

1. 이 프로토타입으로 현장 촬영 흐름과 기록 형식을 먼저 확정합니다.
2. 실제 사진을 모아 라벨링 기준을 정합니다.
3. YOLO 계열 탐지 모델 또는 밀도 추정 모델을 학습합니다.
4. TensorFlow Lite 또는 ONNX Runtime Web으로 스마트폰 앱 내부 추론을 붙입니다.
5. 수동 보정 화면을 추가해 연구자가 결과를 검수하고 모델 개선 데이터로 되돌립니다.
